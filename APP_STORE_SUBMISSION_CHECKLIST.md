## üÜò Common Rejection Reasons

### 1. Guideline 2.1 - App Completeness
- **Issue**: App crashes, incomplete functionality
- **Fix**: Test thoroughly, fix all bugs

### 2. Guideline 4.0 - Design
- **Issue**: Poor UI/UX, placeholder content
- **Fix**: Polish UI, remove placeholders

### 3. Guideline 5.1.1 - Privacy
- **Issue**: Missing privacy policy, unclear data collection
- **Fix**: Add privacy policy, clarify data usage

### 4. Guideline 2.5.1 - Software Requirements
- **Issue**: App doesn't work on all supported devices
- **Fix**: Test on all device sizes

### 5. Guideline 3.1.1 - In-App Purchase
- **Issue**: Payment issues (if applicable)
- **Fix**: Fix payment flow
